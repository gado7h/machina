# Problem Solving

Use this reference when a task benefits from a structured CS workflow.

## Problem Framing

- Identify the exact question being asked.
- List the given constraints and any implied assumptions.
- Distinguish required guarantees from optional optimizations.

## Solution Workflow

- Start with a simple correct approach.
- Identify the bottleneck.
- Improve the bottleneck only if the constraints require it.
- Re-check correctness after each optimization.

## Debugging Workflow

- Build the smallest failing example.
- Compare expected and actual state transitions.
- Check indexing, boundary conditions, mutation, and stale state.
- Verify whether the bug is logical, structural, or caused by misunderstood requirements.

## Explanation Workflow

- Define the concept in one sentence.
- Walk through one representative example.
- Connect the example back to the general rule or invariant.
